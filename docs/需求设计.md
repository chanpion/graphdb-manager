# 图数据库管理系统 - 需求设计文档

## 1. 项目背景

图数据库管理系统是一个企业级的统一图数据库管理平台，旨在解决多种图数据库（Neo4j、NebulaGraph、JanusGraph）的管理难题。系统采用前后端分离架构，支持图数据库连接管理、图实例发现、Schema管理、数据建模、数据CRUD、可视化展示及CSV导入等核心功能。

## 2. 目标用户

- **图数据库管理员**：负责管理多个图数据库实例，进行数据建模和运维管理
- **数据分析师**：需要查询和分析图数据，进行数据探索和可视化展示
- **开发人员**：需要操作图数据库，进行数据的增删改查和导入导出
- **架构师**：需要设计和维护图的Schema结构，定义节点类型、边类型和属性

## 3. 核心功能需求

### 3.1 连接管理

#### 功能描述
支持创建、编辑、删除、测试图数据库连接，支持Neo4j、NebulaGraph、JanusGraph三种图数据库类型。

#### 详细需求

**3.1.1 连接配置**
- 支持以下图数据库类型：
  - **Neo4j**：社区版
  - **NebulaGraph**：标准版
  - **JanusGraph**：标准版
- 连接参数配置：
  - 连接名称（必填，最大长度100字符）
  - 数据库类型选择（下拉选择）
  - 主机地址（必填，IP或域名）
  - 端口号（必填，数字）
  - 用户名（可选）
  - 密码（可选，加密存储）
  - 数据库名/图空间（根据数据库类型显示不同标签）
    - Neo4j显示为"Database"
    - NebulaGraph显示为"Space"
    - JanusGraph显示为"Graph"
  - JanusGraph存储配置（仅JanusGraph显示）：
    - 存储类型选择：HBase / Cassandra
    - 存储配置参数（如HBase的ZooKeeper地址、Cassandra的集群配置）
  - 其他连接参数（JSON格式，高级配置）

**3.1.2 连接操作**
- **创建连接**：填写连接配置表单，点击测试按钮验证连接，成功后保存
- **编辑连接**：修改已有连接的配置信息，重新测试并保存
- **删除连接**：删除连接配置，同时删除相关联的图实例信息
- **测试连接**：点击测试按钮，验证连接参数是否正确，显示连接状态
  - 测试成功：显示绿色提示，显示版本信息、图实例列表
  - 测试失败：显示红色错误信息，说明失败原因

**3.1.3 连接状态管理**
- 实时状态显示：每个连接显示当前状态（已连接、未连接、连接中）
- 自动重连机制：连接断开后自动尝试重新连接
- 连接列表展示：卡片式布局，显示连接名称、类型、主机、端口、状态

### 3.2 图实例管理

#### 功能描述
自动发现并管理图数据库实例中的已有图列表及其Schema信息，支持Neo4j单图、NebulaGraph和JanusGraph多图模式。

#### 详细需求

**3.2.1 图发现功能**
- **图列表获取**：
  - 连接到图数据库后，自动扫描并获取所有图实例
  - Neo4j：获取Database列表（社区版通常只有一个Database）
  - NebulaGraph：获取Space列表（支持多个Space）
  - JanusGraph：获取Graph列表（支持多个Graph）
- **图信息展示**：
  - 图名称
  - 图类型（Database/Space/Graph）
  - 节点数量统计
  - 边数量统计
  - 创建时间
  - 最后更新时间
  - 图状态（正常/异常）

**3.2.2 多图支持差异**
- **Neo4j单图模式**：
  - 每个连接对应一个Database
  - 连接时指定Database名称
  - 直接进入该Database的数据操作界面
- **NebulaGraph多图模式**：
  - 每个连接可管理多个Space
  - 连接后显示Space列表
  - 支持Space切换操作
  - 可在现有Space中操作数据，也可创建新Space
- **JanusGraph多图模式**：
  - 每个连接可管理多个Graph实例
  - 连接后显示Graph列表
  - 支持Graph切换操作
  - 支持HBase或Cassandra作为存储后端
  - 可在现有Graph中操作数据，也可创建新Graph

**3.2.3 图操作功能**
- **创建图实例**：
  - NebulaGraph：创建新Space
  - JanusGraph：创建新Graph
  - Neo4j：社区版不支持创建新Database（提示使用企业版或通过label区分逻辑图）
- **删除图实例**：
  - 删除指定图实例
  - 删除前显示警告提示
  - 删除前确认数据量（节点数、边数）
- **切换图实例**：
  - 在多图模式下，点击切换到其他图实例
  - 切换时刷新Schema信息
  - 切换时清空当前数据视图

### 3.3 Schema管理

#### 功能描述
管理图的Schema定义，包括节点类型、边类型、属性定义、索引的增删改查操作。

#### 详细需求

**3.3.1 节点类型管理**
- **节点类型查询**：
  - 获取图中所有节点类型（Tag/Label）列表
  - 显示每个类型的节点数量
- **创建节点类型**：
  - 定义节点类型名称
  - 定义节点属性（属性名、数据类型、是否必填、默认值）
  - 支持添加多个属性
- **编辑节点类型**：
  - 修改节点类型名称
  - 添加、删除、修改属性定义
- **删除节点类型**：
  - 删除节点类型及其所有属性定义
  - 删除前检查是否有关联节点数据

**3.3.2 边类型管理**
- **边类型查询**：
  - 获取图中所有边类型列表
  - 显示每个类型的边数量
  - 显示边类型对应的源节点类型和目标节点类型约束（如有）
- **创建边类型**：
  - 定义边类型名称
  - 定义边属性（属性名、数据类型、是否必填、默认值）
  - 定义边的起点类型约束（可选）
  - 定义边的终点类型约束（可选）
- **编辑边类型**：
  - 修改边类型名称
  - 添加、删除、修改属性定义
- **删除边类型**：
  - 删除边类型及其所有属性定义
  - 删除前检查是否有关联边数据

**3.3.3 索引管理**
- **索引查询**：
  - 获取图中所有索引列表
  - 显示索引类型、索引字段、索引状态
- **创建索引**：
  - 选择节点类型或边类型
  - 选择要索引的属性
  - 设置索引类型（唯一索引、普通索引）
- **删除索引**：
  - 删除指定索引

### 3.4 数据建模

#### 功能描述
提供可视化的数据建模界面，通过拖拽方式定义节点类型和边类型的关系。

#### 详细需求

**3.4.1 建模画布**
- **可视化编辑区**：
  - 显示节点类型和边类型的可视化关系图
  - 节点类型以矩形展示，显示类型名称和属性数量
  - 边类型以箭头连线展示，显示类型名称
  - 支持拖拽调整元素位置
- **工具栏操作**：
  - 添加节点类型按钮
  - 添加边类型按钮
  - 保存按钮（保存Schema到图数据库）
  - 导出Schema按钮（导出为JSON格式）
  - 刷新按钮（从图数据库重新加载Schema）

**3.4.2 属性编辑面板**
- **选中元素编辑**：
  - 点击节点类型或边类型，右侧显示属性编辑面板
  - 显示类型名称（可编辑）
  - 显示属性列表
- **属性管理**：
  - 添加属性：弹出属性添加对话框
    - 属性名称
    - 数据类型选择：String, Integer, Long, Double, Boolean, Date
    - 是否必填（单选）
    - 默认值（根据类型输入）
  - 编辑属性：修改属性定义
  - 删除属性：删除指定属性
  - 上移、下移：调整属性顺序

**3.4.3 关系定义**
- **边类型关系配置**：
  - 为边类型定义起点类型约束
  - 为边类型定义终点类型约束
  - 支持多种起点/终点类型（多选）

### 3.5 数据CRUD操作

#### 功能描述
提供节点和边的增删改查功能，所有节点和边包含唯一uid属性。

#### 详细需求

**3.5.1 数据模型规范**
- **节点数据模型（Vertex）**：
  - uid：唯一标识符，字符串类型，必填
  - tagName：节点类型名称，关联Schema定义
  - properties：属性集合，Map<String, Object>类型
  - createdAt：创建时间，时间戳
  - updatedAt：更新时间，时间戳

- **边数据模型（Edge）**：
  - uid：唯一标识符，字符串类型，必填
  - edgeType：边类型名称，关联Schema定义
  - sourceUid：起始节点uid，必填
  - targetUid：目标节点uid，必填
  - properties：属性集合，Map<String, Object>类型
  - createdAt：创建时间，时间戳
  - updatedAt：更新时间，时间戳

**3.5.2 节点CRUD**
- **查询节点**：
  - 列表查询：分页显示节点列表
  - 筛选条件：按节点类型、按属性值筛选
  - 搜索功能：按uid或属性值模糊搜索
  - 排序功能：按创建时间、按属性值排序
- **创建节点**：
  - 选择节点类型
  - 填写属性值（根据Schema动态生成表单）
  - 系统自动生成uid（可手动修改）
  - 保存验证：必填属性检查、数据类型检查
- **编辑节点**：
  - 修改节点属性值
  - 修改节点类型（可选）
  - 更新updatedAt时间
- **删除节点**：
  - 通过uid删除节点
  - 级联删除：删除节点时，是否删除关联边（可配置）
  - 删除前确认：显示将影响的边数量
- **批量操作**：
  - 批量删除：选中多个节点，批量删除
  - 批量导出：选中多个节点，导出为CSV/JSON

**3.5.3 边CRUD**
- **查询边**：
  - 列表查询：分页显示边列表
  - 筛选条件：按边类型、按起始节点、按目标节点筛选
  - 搜索功能：按uid或属性值模糊搜索
  - 排序功能：按创建时间排序
- **创建边**：
  - 选择边类型
  - 选择起始节点（通过节点选择器或输入uid）
  - 选择目标节点（通过节点选择器或输入uid）
  - 填写属性值（根据Schema动态生成表单）
  - 系统自动生成uid（可手动修改）
  - 保存验证：起始节点和目标节点必须存在、必填属性检查
- **编辑边**：
  - 修改边属性值
  - 修改边类型（可选）
  - 修改起始节点或目标节点（可选）
  - 更新updatedAt时间
- **删除边**：
  - 通过uid删除边
  - 删除前确认：显示边详细信息
- **批量操作**：
  - 批量删除：选中多个边，批量删除
  - 批量导出：选中多个边，导出为CSV/JSON

### 3.6 图可视化

#### 功能描述
使用D3.js实现交互式图数据可视化展示，支持缩放、拖拽、节点展开等操作。

#### 详细需求

**3.6.1 基础可视化**
- **力导向布局**：
  - 默认使用D3力导向算法布局
  - 节点之间的连线根据边的连接关系自动生成
  - 支持布局参数调整（斥力强度、连线距离）
- **节点渲染**：
  - 节点以圆形展示
  - 节点颜色：根据节点类型使用不同颜色
  - 节点大小：根据节点的度数或属性值动态调整
  - 节点标签：显示节点名称或关键属性
- **边渲染**：
  - 边以直线或曲线展示
  - 边颜色：根据边类型使用不同颜色
  - 边粗细：可配置
  - 边标签：显示边类型名称或关键属性
  - 边方向：有向边显示箭头

**3.6.2 交互功能**
- **节点交互**：
  - 单击节点：选中节点，高亮显示，显示属性面板
  - 双击节点：进入节点详情页或展开邻居节点
  - 拖拽节点：拖动节点位置，重新布局
  - 右键节点：显示上下文菜单（编辑、删除、展开邻居、查看属性）
- **边交互**：
  - 单击边：选中边，高亮显示，显示属性面板
  - 双击边：进入边详情页
  - 右键边：显示上下文菜单（编辑、删除、查看属性）
- **画布交互**：
  - 滚轮缩放：以鼠标位置为中心缩放
  - 拖拽平移：按住鼠标拖动画布
  - 缩放控件：放大、缩小、适应屏幕、重置视图
  - 布局切换：力导向布局、圆形布局、分层布局、网格布局

**3.6.3 高级可视化功能**
- **节点展开**：
  - 单击节点后展开其邻居节点
  - 显示展开层级（1度、2度、3度邻居）
  - 可折叠已展开的节点
- **路径高亮**：
  - 选中两个节点，高亮显示最短路径
  - 显示路径上的所有节点和边
- **子图展示**：
  - 选中节点，显示其k度邻居子图
  - 可设置k值（1-10）
- **群组显示**：
  - 按节点类型分群组显示
  - 可折叠/展开群组

**3.6.4 图例和统计**
- **图例**：
  - 显示节点类型图例（颜色说明）
  - 显示边类型图例（颜色和线条说明）
  - 支持点击图例筛选对应类型的节点/边
- **统计信息**：
  - 显示节点总数
  - 显示边总数
  - 显示当前视图中的节点数和边数
  - 显示当前缩放比例

### 3.7 CSV导入功能

#### 功能描述
支持从CSV文件批量导入节点和边数据到图数据库。

#### 详细需求

**3.7.1 文件上传**
- **上传方式**：
  - 点击上传按钮选择文件
  - 拖拽文件到上传区域
- **文件格式支持**：
  - CSV格式（.csv文件）
  - UTF-8编码
  - 支持大文件（建议<100MB）
- **文件预览**：
  - 上传后预览CSV前10行数据
  - 显示表格：行号、所有列数据
  - 识别分隔符（逗号、分号、Tab）

**3.7.2 字段映射**
- **节点CSV映射**：
  - 选择"节点"导入模式
  - CSV列选择：选择uid列（必填）、tagName列、属性列
  - 属性映射：将CSV列映射到节点属性
  - 映射验证：检查必填字段是否已映射
- **边CSV映射**：
  - 选择"边"导入模式
  - CSV列选择：选择uid列（必填）、edgeType列、sourceUid列、targetUid列、属性列
  - 属性映射：将CSV列映射到边属性
  - 映射验证：检查必填字段是否已映射

**3.7.3 导入配置**
- **导入目标**：
  - 选择目标图实例（从图列表中选择）
  - 创建新图选项：输入新图名称（适用于创建图并导入）
- **导入选项**：
  - 批量大小：每批处理的数据量（100/500/1000）
  - 错误处理策略：跳过错误记录 / 停止导入 / 回滚导入
  - 重复数据处理：覆盖 / 跳过 / 合并
- **数据验证**：
  - Schema验证：验证数据是否符合Schema定义
  - 类型验证：验证属性值的数据类型
  - 必填验证：验证必填字段是否为空
  - 引用验证：验证边的sourceUid和targetUid是否存在

**3.7.4 导入执行**
- **进度显示**：
  - 显示导入进度条
  - 显示已处理行数 / 总行数
  - 显示成功数、失败数
  - 显示错误信息列表（前50条）
- **导入结果**：
  - 显示导入统计：成功节点数、失败节点数、成功边数、失败边数
  - 提供错误日志下载：下载导入失败的详细错误信息
  - 提供继续导入：部分失败后可选择继续导入有效数据

### 3.8 数据导出功能

#### 功能描述
支持将图数据导出为CSV或JSON格式。

#### 详细需求

**3.8.1 导出格式**
- **CSV导出**：
  - 支持节点数据导出
  - 支持边数据导出
  - 支持全图导出（节点+边）
  - 支持自定义导出字段（选择要导出的属性）
- **JSON导出**：
  - 导出完整图数据（Schema + 节点 + 边）
  - 导出选中的节点和边
  - 支持压缩导出（gzip）
- **图片导出**：
  - 导出当前可视化视图为PNG/JPG图片
  - 导出全图数据为SVG矢量图
  - 支持自定义图片分辨率和背景色

### 3.9 原生查询功能

#### 功能描述
支持通过图数据库的原生查询语句查询图数据，包括Neo4j的Cypher、NebulaGraph的nGQL、JanusGraph的Gremlin，提供友好的查询编辑器和结果展示界面。

#### 详细需求

**3.9.1 查询编辑器**
- **编辑器界面**：
  - 代码编辑器：支持语法高亮、代码补全、括号匹配
  - 行号显示：显示查询语句的行号
  - Tab支持：支持Tab缩进（默认4空格）
  - 快捷键支持：
    - Ctrl + Enter：执行查询
    - Ctrl + /：注释/取消注释选中行
    - Ctrl + S：保存查询（可选功能）
- **查询语言支持**：
  - 根据当前连接的图数据库类型自动切换查询语言
  - Neo4j：Cypher查询语言
  - NebulaGraph：nGQL查询语言
  - JanusGraph：Gremlin查询语言
- **查询模板**：
  - 提供常用查询模板
  - 模板分类：
    - 基础查询（节点查询、边查询、路径查询）
    - 聚合查询（计数、求和、分组）
    - 图算法（最短路径、PageRank、社区发现）
  - 点击模板自动填入编辑器
- **查询历史**：
  - 自动保存执行过的查询语句
  - 显示最近20条查询历史
  - 点击历史记录快速加载查询
  - 支持清空历史记录
  - 显示查询执行时间

**3.9.2 查询执行**
- **执行方式**：
  - 点击"执行查询"按钮
  - 快捷键：Ctrl + Enter
  - 查询执行时显示加载动画
- **查询超时**：
  - 默认超时时间：60秒
  - 可配置超时时间（30秒/60秒/120秒/300秒）
  - 超时后自动取消查询并提示用户
- **查询限制**：
  - 单次查询返回结果集限制：默认1000条
  - 可配置结果集限制（100/500/1000/5000）
  - 超出限制时显示提示并提供分页加载
- **查询安全**：
  - 防止注入攻击：参数化查询、输入验证
  - 限制查询复杂度：限制查询深度、限制返回数据量
  - 查询审计：记录所有执行的查询语句

**3.9.3 结果展示**
- **表格展示**：
  - 表格形式展示查询结果
  - 自动识别列名和数据类型
  - 支持列排序：点击列头进行升序/降序排序
  - 支持列筛选：显示/隐藏特定列
  - 支持列宽度调整：拖拽列边界调整宽度
- **结果统计**：
  - 显示查询执行时间
  - 显示返回结果数量
  - 显示查询状态（成功/失败/超时）
  - 显示影响行数（对于写操作）
- **结果导出**：
  - 导出为CSV格式
  - 导出为JSON格式
  - 导出为Excel格式（可选功能）
- **特殊数据类型展示**：
  - 节点数据：显示为"节点(uid, tagName)"格式
  - 边数据：显示为"边(uid, edgeType, sourceUid → targetUid)"格式
  - 数组/列表数据：支持展开/折叠查看
  - 地理坐标数据：显示为"(lat, lon)"格式

**3.9.4 可视化结果**
- **结果可视化**：
  - 支持将查询结果转换为图可视化
  - 自动识别节点和边数据
  - 使用D3.js渲染结果图
  - 支持拖拽、缩放、节点展开等交互
- **可视化选项**：
  - 选择要可视化的节点和边类型
  - 设置节点数量限制（避免性能问题）
  - 选择布局算法（力导向/圆形/分层）
- **可视化导出**：
  - 导出当前可视化视图为PNG图片
  - 导出全图为SVG矢量图

**3.9.5 查询优化建议**
- **查询分析**：
  - 分析查询语句的执行计划
  - 显示查询性能瓶颈
  - 提供查询优化建议
- **性能指标**：
  - 显示查询扫描的节点数量
  - 显示查询扫描的边数量
  - 显示查询使用的索引
  - 显示查询的内存使用情况
- **优化建议**：
  - 建议创建缺失的索引
  - 建议优化查询语句结构
  - 建议使用更高效的查询模式

**3.9.6 批量查询**
- **批量执行**：
  - 支持批量执行多条查询语句
  - 语句之间用分号";"分隔
  - 依次执行并显示每条查询的结果
- **查询脚本**：
  - 支持上传和执行查询脚本文件
  - 支持下载查询结果为报告
  - 支持查询脚本管理（保存、加载、编辑）

## 6. 系统约束

- **技术约束**：
  - 前端使用Vue 3 + JavaScript
  - 前端UI使用Element Plus
  - 可视化使用D3.js
  - 后端使用Spring Boot
  - 元数据存储使用MySQL
  - 不使用其他前端框架（React、Angular等）
- **图数据库约束**：
  - Neo4j：支持社区版和企业版
  - NebulaGraph：支持标准版
  - JanusGraph：支持HBase或Cassandra存储后端
- **架构约束**：
  - 图操作和图数据操作必须完全分离
  - 所有节点和边必须包含唯一uid属性
  - 支持多图模式（NebulaGraph/JanusGraph）
- **部署约束**：
  - 本地部署环境
  - 不依赖云服务
  - 支持Docker部署（可选）

  - 查看连接和图实例（只读）

  - 数据导出
- **开发人员（Developer）**：
  - 连接管理和图管理
  - 数据CRUD操作
  - 数据建模操作
  - 数据导入导出

### 7.2 权限控制

- **连接权限**：
  - 创建连接：Admin和Developer
  - 编辑连接：Admin和Developer
  - 删除连接：仅Admin
- **图权限**：
  - 创建图：Admin和Developer
  - 删除图：仅Admin
  - Schema管理：Admin和Developer

  - 数据查询：所有角色
  - 数据CRUD：Admin和Developer
  - 数据导入：Admin和Developer
  - 数据导出：所有角色

## 8. 数据质量要求

### 8.1 数据完整性

- **UID唯一性**：
  - 所有节点uid必须在图中唯一
  - 所有边uid必须在图中唯一
  - 导入时自动检测重复uid并提示
- **引用完整性**：
  - 边的sourceUid必须引用存在的节点
  - 边的targetUid必须引用存在的节点
  - 删除节点时检查引用完整性

### 8.2 数据一致性

- **Schema一致性**：
  - 节点必须属于已定义的节点类型
  - 边必须属于已定义的边类型
  - 属性值必须符合Schema定义的数据类型
- **时间一致性**：
  - 创建时间自动设置为当前时间
  - 更新时间自动更新为当前时间
  - 时间字段不可手动修改
